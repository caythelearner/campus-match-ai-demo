更新时间：2026-06-07

当前 demo 地址：

```text
http://localhost:8020/demo/index.html
```

当前项目总占用约 1.8G，主要是 `.venv-lite` 运行环境和 `.hf-models` 本地文本 embedding 模型。

## 1. 项目定位

本项目是一个校园 AI 匹配产品 demo，包含两个视角：

1. 用户视角：模拟真实 dating app / 搭子 app 的使用体验。
2. 管理员视角：展示知识图谱、向量检索、GraphRAG、GNN、Neo4j、FAISS、治理风控等课程方法。

产品核心包括：

1. 心动星球：偏恋爱匹配，强调三观、性格、未来规划、匹配理由。
2. 闪电搭子：偏即时搭子，强调地点、时间、具体任务、安全和信用。
3. 雷达广场：搜索、Tag、语义联想、寻人启事墙。
4. 消息页：聊天、AI 僚机、情绪气象站、共鸣实验室。
5. 档案页：灵魂卡片、标签证据、记忆博物馆、星际信用。

## 3. 用户端已经实现的内容

### 3.1 星球入口

星球页包含两个模式：

1. 心动星球：查看推荐对象、匹配理由、共同兴趣、共同价值观、风险提示。
2. 闪电搭子：查看即时搭子请求，加入饭搭子、运动搭子、学习搭子等场景。

“说一句今天想做的事”现在不是装饰输入框，而是会做意图路由：

| 用户输入倾向 | 路由到 |
|---|---|
| 吃饭、运动、自习、拍照、搭子 | 雷达 / 闪电搭子 |
| 聊天、破冰、认识一下、怎么回复 | 消息 |
| 见面、约、首约、线下 | 首约策划 |
| 反馈、见后感受、不舒服、还不错 | 档案 / 记忆博物馆 |
| 推荐、心动、对象、看看谁合适 | 星球 / 心动星球 |

页面会展示“今日意图”、置信度、识别原因和后续动作，避免用户输入后不知道有什么作用。

### 3.2 雷达广场

雷达页实现了：

1. 搜索框。
2. 热门 Tag 筛选。
3. AI 语义联想说明。
4. 寻人启事墙。
5. 点击候选人进入聊天。
6. 场景化匹配结果展示。

当前搜索和筛选是前端基于生成数据、标签、画像文本和匹配结果做的本地计算，不依赖外部 API。

### 3.3 消息与聊天

消息页实现了：

1. 用户自由输入。
2. 对方回复。
3. 每天最多 3 轮聊天，避免无限点击刷数据。
4. AI 建议来源展示。
5. 实时聊天 RAG trace 展示。
6. 情绪气象站提醒。
7. 共鸣实验室破冰建议。

当前聊天回复逻辑：

```text
用户输入
-> 浏览器端 hash embedding
-> 检索画像 / 匹配理由 / 破冰库 / 安全库 / 场景库
-> Top-K 结果
-> 根据命中文档生成回复
-> 展示检索来源和 query
```

它已经不是单纯固定模板，但也不是在线 LLM API 生成。当前属于“本地轻量 RAG + 规则生成”。

已经加入的聊天知识库包括：

1. 共同兴趣开场。
2. 具体活动推荐。
3. 低压力首约。
4. 学习搭子。
5. 低能量照顾。
6. 边界感提醒。
7. 运动搭子成局。
8. 饭搭子回复。
9. 拍照 / Citywalk。
10. 考试压力安抚。
11. 见后反馈。

最近修过的问题：

1. 删除了“我看到你也对 X 感兴趣，最近有没有相关活动推荐？”这种重复套话。
2. 删除了“我也注意到我们都提到了 X”的固定前缀。
3. 增加了最近回复去重，避免连续生成同一种句式。
4. 对“剧本杀”这类高频兴趣增加专门变体，会问偏好、雷点、玩法、时长，而不是反复复述标签。

### 8.2 当前是规则 / 合成模拟

1. 用户数据是合成问卷数据，不是真实校园用户。
2. 图片是占位图和 prompt，不是真人照片。
3. 情绪气象站指标是合成聊天统计，不是真实聊天模型分析。
4. LBS、天气、校区安全是模拟上下文，不是真实 API。
5. 聊天回复是本地 RAG + 规则生成，不是在线 LLM。
6. GraphRAG 解释是模板生成，不是调用大模型生成。
7. 首约地点和时间是基于模拟数据生成。

2. Stable Diffusion / Midjourney 图片生成 API。
3. 真实天气 API。
4. 真实 LBS / 地图 API。
